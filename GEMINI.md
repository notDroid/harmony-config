# Config Repository Agents Guide

This repository manages environment-specific configurations and deployments for Harmony Chat. It follows GitOps principles, acting as the environment-specific source of truth while the `app-repo` remains environment-agnostic.

## Stack
- **Task** (Taskfile.yaml)
- **Helmfile** (for templating manifests)
- **ArgoCD** (for GitOps continuous deployment)
- **SOPS** (for secrets management)
- **Kubernetes / Kind** (for local E2E testing)
- **Terraform / Terragrunt** (for AWS provisioning)

## Commands
```bash
# Secrets Management
task secrets:setup     # Configure .sops.yaml with new age key
task secrets:generate  # Generate a new secrets.yaml file for the target K8S_ENV
task secrets:encrypt   # Encrypt secrets.yaml in place using SOPS
task secrets:decrypt   # Decrypt secrets.yaml in place using SOPS

# Local E2E Testing (ArgoCD on Kind)
task argocd:install    # Installs ArgoCD and creates namespaces

# Infrastructure & OCI Pipeline
task infra:fetch       # Fetches agnostic k8s components into .cache/k8s based on K8S_VERSION in config.yaml
task infra:template    # Templates Helmfile to standard Kubernetes manifests in dist/
task infra:publish     # Packages rendered manifests and pushes them to the OCI registry

# Cleanup
task clean             # Cleans up the dist/ and .cache/ directories
```

## Hard Limits
- **NEVER** commit unencrypted secrets. Always use SOPS for `secrets.yaml` files.
- **NEVER** place environment-specific logic or production values in the `app-repo`. All staging/production values, secrets, and environment overrides MUST live here in the `config-repo`.
- **NEVER** edit files in `.cache/k8s` or `dist/` directly. These are dynamically generated by `task infra:fetch` and `task infra:template`.

## Conventions
- **GitOps Separation:** 
  - The **App Repo** handles application source code, Docker image building, and generic infrastructure templates (as well as local-only `docker-compose` setups). It maintains zero awareness of downstream configurations.
  - This **Config Repo** contains all specific configurations, SOPS-encrypted secrets, and triggers for deployments. It utilizes a high-level `values.yaml` and `secrets.yaml` to control the entire deployment.
- **Dynamic Configuration:** The `config.yaml` file acts as the primary configuration for the project's global settings (project name, k8s version to fetch, registry URLs). The `Taskfile.yaml` dynamically populates its variables from this file using `yq`.
- **Helmfile Templating Strategy:** We use Helmfile to render standard Kubernetes manifests (`task infra:template`) before packaging them as OCI artifacts (`task infra:publish`). The `HELMFILE_VALUES` combine generic values from the fetched k8s components (in `.cache/k8s/`) with environment-specific overrides defined in `environments/<env>/`.

## Gotchas
- **Ephemeral Cache:** The `.cache/k8s/` folder contains a downloaded snapshot of the `app-repo`'s k8s directory based on the `K8S_VERSION` defined in `config.yaml`. Do not try to make persistent changes here; update the `app-repo` instead.
- **Local Images:** For local E2E testing using kind, the `app-repo` must build and load the Docker images into the cluster. This is typically done in the `app-repo` via `task image:load:kind` (which builds `harmony-backend:local` and `init-dynamodb:latest` and loads them into kind).

## Structure Pointers
- `config.yaml`: Global configuration file (read by Taskfile).
- `environments/<env>/`: The absolute source of truth for environment-specific values and secrets.
  - `environments/local/`: Values, secrets, and ArgoCD definitions for the local Kind cluster.
  - `environments/staging/`: Values, secrets, and overrides for the remote staging environment.
- `.cache/k8s/`: Ephemeral directory containing fetched k8s components.
- `dist/`: Output directory for rendered Kubernetes manifests.
- `terraform/`: Contains Terragrunt configurations for provisioning AWS infrastructure.
